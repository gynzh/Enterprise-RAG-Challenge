# RAG Challenge 复现结果可视化系统

## 📝 概述

这是一个基于Gradio构建的可视化系统，用于展示RAG Challenge获胜方案的复现结果。通过直观的Web界面，您可以：

- 📄 在线预览PDF文档
- 📊 查看各阶段处理统计
- 🔍 深入分析JSON数据结构
- 💬 比较不同配置的问答结果

## 🏗️ 系统架构

```
visualization/
├── app.py                 # 主应用入口
├── run.py                 # 启动脚本
├── requirements_viz.txt   # 依赖清单
├── README.md             # 说明文档
└── components/           # 核心组件
    ├── __init__.py
    ├── data_loader.py    # 数据加载器
    ├── pdf_viewer.py     # PDF查看器
    ├── json_viewer.py    # JSON数据展示
    └── stats_viewer.py   # 统计图表
```

## 🚀 快速开始

### 1. 安装依赖

```bash
# 在 visualization 目录下
cd visualization
pip install -r requirements_viz.txt
```

### 2. 确保数据完整

请确保以下数据目录存在且包含处理结果：

```
../data/test_set/
├── pdf_reports/                    # 原始PDF文件
│   ├── 194000c9109c6fa628f1fed33b44ae4c2b8365f4.pdf
│   └── 9d7a72445aba6860402c3acce75af02dc045f74d.pdf
├── debug_data/                     # 各阶段处理结果
│   ├── 01_parsed_reports/          # PDF解析结果
│   ├── 02_merged_reports/          # 合并报告
│   └── 03_reports_markdown/        # Markdown格式
├── databases/                      # 数据库文件
│   ├── chunked_reports/            # 切块数据
│   └── vector_dbs/                 # 向量数据库
└── answers_*.json                  # 问答结果
```

### 3. 启动应用

```bash
# 直接运行启动脚本
python run.py

```

### 4. 访问界面

启动成功后，访问: http://localhost:7860

## 🎛️ 界面功能

### 顶部控制面板

- **📄 文档选择**: 选择要查看的PDF报告
- **⚙️ 配置选择**: 选择问答结果的配置版本（如base、gemini_thinking_fc等）
- **🔄 刷新按钮**: 重新加载所有数据

### 左侧PDF预览区

- **📄 在线预览**: 直接在浏览器中查看PDF文档
- **🔍 缩放控制**: 支持放大、缩小、重置
- **⛶ 全屏模式**: 支持全屏查看
- **📥 下载功能**: 支持下载PDF文件

### 右侧标签页

#### 1. 📄 PDF解析
- **📊 解析统计**: 页数、表格数量、文本长度等统计信息
- **🔄 处理状态**: 各阶段处理完成情况
- **🌳 JSON结构**: 解析结果的完整数据结构
- **📊 表格概览**: 表格数据的详细分析

#### 2. 🔄 序列化
- **📝 Markdown展示**: 序列化后的报告内容
- **🔗 合并数据**: 合并后的JSON数据结构
- **📊 序列化统计**: 表格序列化前后对比

#### 3. 💾 数据注入
- **🧩 切块统计**: 切块数量、Token统计、页面覆盖
- **📈 分布图表**: 切块在各页面的分布情况
- **💾 向量数据库**: FAISS数据库大小和状态
- **🔍 切块详情**: 具体切块数据的JSON展示

#### 4. ❓ 问答结果
- **📊 结果统计**: 问题数量、回答完成率、引用统计
- **💬 问答展示**: 详细的问答对展示
- **🔗 引用信息**: 答案的页面引用情况
- **⚙️ 配置对比**: 不同配置版本的结果对比

## 🎨 界面设计

### 设计理念

根据用户偏好[[memory:2528750]]，界面采用iOS扁平化设计风格，配色方案为：

- **主色调**: 清新的浅蓝色（Apple主题色）
- **背景色**: 明亮的白色和浅灰色
- **强调色**: 清新明亮的蓝色系
- **文字**: 深灰色，确保可读性

### 响应式布局

- **桌面端**: 左右分栏，PDF预览占50%，结果展示占50%
- **平板端**: 自适应缩放，保持布局结构
- **手机端**: 垂直堆叠，滚动查看

## 🔧 自定义和扩展

### 添加新的数据展示

1. 在 `components/` 目录下创建新组件
2. 在 `data_loader.py` 中添加数据加载方法
3. 在 `app.py` 中集成新组件

### 修改界面样式

- 编辑各组件中的CSS样式
- 修改 `app.py` 中的Gradio主题
- 调整布局比例和组件配置

### 添加新的配置支持

- 修改 `data_loader.py` 中的配置扫描逻辑
- 更新文件命名模式匹配
- 添加相应的数据处理方法

## 🐛 故障排除

### 常见问题

1. **PDF无法显示**
   - 检查浏览器PDF插件是否启用
   - 尝试使用Chrome或Firefox浏览器
   - 使用下载链接作为备用方案

2. **数据加载失败**
   - 确认数据目录路径正确
   - 检查文件权限
   - 查看控制台错误信息

3. **依赖安装问题**
   - 使用虚拟环境隔离依赖
   - 更新pip到最新版本
   - 检查Python版本兼容性

### 调试模式

```bash
# 启用详细日志
export GRADIO_DEBUG=1
python run.py
```
